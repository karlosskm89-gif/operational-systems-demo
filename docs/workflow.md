# Booking Workflow

This demo shows a simple operational flow:

1. A visitor submits a public booking request.
2. The request is stored as demo JSON data.
3. The visitor receives a clear confirmation state.
4. Admin users can view the booking queue.
5. Each booking has a simple status workflow.
6. Bookings can be exported as CSV.

This is a public-safe portfolio demo, not production client code.

## Why this matters

Many small business websites capture enquiries but provide little operational structure afterwards.

This demo shows the next layer:

- request capture
- admin visibility
- workflow status
- exportability
- simple operational reporting

That is the ASR angle: useful systems behind the public website.
