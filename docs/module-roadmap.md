# Module Roadmap

This repo currently contains one complete operational module:

## Current Module

### Booking Workflow

Includes:
- public intake form
- confirmation state
- admin queue
- detail view
- status handling
- CSV export

## Future Modules

Skeleton modules now included:

### Records Module
Included as a lightweight skeleton with list and detail views.

### Invoice Workflow
Included as a lightweight skeleton with invoice states, detail views and CSV export.

### Admin Dashboard
Summary widgets for queues, statuses and recent workflow activity.

### Notes / Activity Log
Simple operational history for admin-side follow-up.

## Scope Control

This repo should stay believable and focused.

Avoid:
- unnecessary platform complexity
- overbuilt dashboards
- unrealistic multi-tenant logic
- features that distract from the workflow proof

The goal is practical operational proof with intentionally scoped modules.
