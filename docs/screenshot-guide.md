# Screenshot Guide

Recommended screenshot structure:

```text
/screenshots
  /public-flow
  /admin
  /workflow
  /mobile
```

## Required screenshots

### Public Flow

```text
public-home.webp
booking-form.webp
confirmation-state.webp
```

### Admin

```text
admin-dashboard.webp
admin-detail.webp
status-update.webp
csv-export.webp
```

### Workflow

```text
workflow-overview.webp
workflow-mobile.webp
```

### Mobile

```text
mobile-home.webp
mobile-booking-form.webp
mobile-admin-dashboard.webp
```

## Capture rules

Use:

- clean browser screenshots
- realistic demo records
- `.webp` where possible
- consistent crops
- mobile and desktop views

Avoid:

- browser bookmarks/tabs
- private accounts
- real client data
- blurry compression
- giant full-page screenshots unless needed
