# Ecosystem Positioning

This repository is the operational systems layer of the ASR public proof ecosystem.

## Repo 1 — ASR Web Services Starter Kit

Purpose:
- public-facing adaptable website systems
- profile switching
- fictionalised showcase mode
- case study structure
- ASR philosophy and public-safe architecture

## Repo 2 — ASR Operational Systems Demo

Purpose:
- operational workflow proof
- booking and enquiry intake
- admin queue visibility
- status handling
- export-oriented infrastructure
- future operational module foundation

## Why these are separate

The starter kit shows how ASR structures adaptable public-facing systems.

This repo shows how ASR thinks about the operational layer behind those systems.

That separation keeps the ecosystem clear:

- external presentation systems
- internal operational systems

Together, they support ASR’s broader positioning:

> calm, maintainable operational systems for real organisations.
